主题开发指南

基础概念

1. 什么是主题？

主题是定义编辑器视觉外观的配置集合，包括颜色方案、图标样式、语法高亮等。通过开发主题，你可以创建独特的视觉体验。

2. 主题类型

  • 颜色主题：界面配色方案
  • 图标主题：文件和文件夹图标
  • 语法主题：代码高亮规则
  • 产品图标：界面操作图标

开发准备

1. 环境搭建

  • 创建项目：

    # 创建主题项目
yo cursor-theme

# 安装依赖
npm install

  • 项目结构：

    my-theme/
├── themes/             # 主题文件目录
│   ├── dark.json      # 深色主题
│   └── light.json     # 浅色主题
├── package.json       # 配置文件
└── README.md          # 文档

2. 基本配置

  • package.json： 
    {
  "name": "my-theme",
  "version": "1.0.0",
  "contributes": {
    "themes": [
      {
        "label": "My Theme",
        "uiTheme": "vs-dark",
        "path": "./themes/dark.json"
      }
    ]
  }
}

颜色主题开发

1. 界面颜色

  • 基础颜色：

    {
  "colors": {
    "editor.background": "#1E1E1E",
    "editor.foreground": "#D4D4D4",
    "activityBar.background": "#333333",
    "sideBar.background": "#252526",
    "statusBar.background": "#007ACC"
  }
}

  • 语法颜色：

    {
  "tokenColors": [
    {
      "scope": ["comment"],
      "settings": {
        "foreground": "#6A9955"
      }
    },
    {
      "scope": ["string"],
      "settings": {
        "foreground": "#CE9178"
      }
    }
  ]
}

2. 语法高亮

  • 语言标记：

    {
  "tokenColors": [
    {
      "scope": ["keyword.control"],
      "settings": {
        "foreground": "#C586C0",
        "fontStyle": "italic"
      }
    },
    {
      "scope": ["entity.name.function"],
      "settings": {
        "foreground": "#DCDCAA"
      }
    }
  ]
}

  • 语法规则：

    {
  "tokenColors": [
    {
      "scope": [
        "punctuation.definition.string",
        "punctuation.definition.parameters"
      ],
      "settings": {
        "foreground": "#808080"
      }
    }
  ]
}

图标主题开发

1. 文件图标

  • 图标定义：

    {
  "iconDefinitions": {
    "_file": {
      "iconPath": "./icons/file.svg"
    },
    "_folder": {
      "iconPath": "./icons/folder.svg"
    }
  }
}

  • 文件关联：

    {
  "fileExtensions": {
    "js": "_file_js",
    "ts": "_file_ts",
    "json": "_file_json"
  },
  "fileNames": {
    "package.json": "_npm",
    "tsconfig.json": "_tsconfig"
  }
}

2. 文件夹图标

  • 文件夹状态：

    {
  "folderNames": {
    "src": "_folder_src",
    "test": "_folder_test"
  },
  "folderNamesExpanded": {
    "src": "_folder_src_open",
    "test": "_folder_test_open"
  }
}

  • 特殊文件夹：

    {
  "folderNames": {
    "node_modules": "_folder_node_modules",
    ".git": "_folder_git"
  }
}

主题测试

1. 本地测试

  • 调试配置：

    {
  "launch": {
    "configurations": [
      {
        "type": "extensionHost",
        "request": "launch",
        "name": "Launch Theme",
        "args": ["--extensionDevelopmentPath=${workspaceFolder}"]
      }
    ]
  }
}

  • 颜色预览：

    // 使用 Color Picker 预览颜色
// 在颜色值上按 Ctrl+Space
"editor.background": "#1E1E1E" // 预览颜色

2. 主题调试

  • 实时预览：

    {
  "workbench.colorCustomizations": {
    "[My Theme]": {
      "editor.background": "#1E1E1E"
    }
  }
}

  • 调试工具：

    1. 开发者工具 (F12)
2. 主题调色板
3. 作用域检查器

发布和维护

1. 打包发布

  • 发布准备：

    {
  "scripts": {
    "package": "cursor package",
    "publish": "cursor publish"
  }
}

  • 发布检查：

    # 验证主题
npm run test

# 打包主题
npm run package

# 发布主题
npm run publish

2. 版本更新

  • 版本管理：

    {
  "version": "1.0.0",
  "repository": {
    "type": "git",
    "url": "https://github.com/username/my-theme"
  }
}

  • 更新日志：

    # 更新日志

## [1.0.0] - 2024-01-01
- 初始版本发布
- 支持深色和浅色主题
- 添加基本文件图标

最佳实践

1. 设计原则

  • 颜色对比：

    {
  "colors": {
    // 保持足够的对比度
    "editor.foreground": "#D4D4D4",
    "editor.background": "#1E1E1E",
    
    // 突出重要元素
    "editor.selectionBackground": "#264F78"
  }
}

  • 一致性：

    {
  "colors": {
    // 使用一致的色调
    "statusBar.background": "#007ACC",
    "activityBar.background": "#333333",
    "titleBar.activeBackground": "#3C3C3C"
  }
}

2. 性能考虑

  • 图标优化：

    {
  // 使用 SVG 图标
  "iconDefinitions": {
    "_file": {
      "iconPath": "./icons/file.svg"
    }
  }
}

  • 规则优化：

    {
  // 合并相似规则
  "tokenColors": [
    {
      "scope": [
        "keyword.control",
        "keyword.operator"
      ],
      "settings": {
        "foreground": "#C586C0"
      }
    }
  ]
}

小结

主题开发需要注意以下几点：

  1. 注重视觉体验
  2. 保持颜色和谐
  3. 考虑可访问性
  4. 优化性能表现

记住，好的主题应该既美观又实用，在开发过程中要多收集用户反馈，不断改进完善。
